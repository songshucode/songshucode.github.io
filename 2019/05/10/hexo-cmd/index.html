hexo 命令

发表于 2019-05-10 | 更新于 2019-06-18

hexo 使用的常用命令，初始化，建立新章节，添加新的属性，生成静态文件，发布

inistall

git install

官网下载git安装包，一路next

node.js install

官网下载node.js安装包，会默认安装npm

hexo install

使用git cmd或者windows cmd安装

setting ssh key for git

  • generate ssh key

if reach any error, delete the older ssh key file in the .ssh/

  • submit public key

copy the public key in .ssh/id-rsa.pubto the github web

asssociating blog and github

edit the file_config.ymlin the porject directory

operation

init

新建一个网站，使用folder制定文件夹，默认为当前文件夹。

new

新建一篇文章，如果没有设置layout则默认使用_config.yml中的default_layout参数。如果标题中使用空格，需要使用引号括起来

draft

新建一篇草稿，草稿并不会被更新到服务器端的个人博客中，会在本地文件夹source\_drafts中新建一个md文件

publish

公布草稿为博客

page

为博客添加新的属性（tag,…）

会在相应在source文件夹下生成相应的markdown文件，分别在每个文件头处添加：

generate

生成静态文件

选项 描述
-d 文件生成后部署到网站上
-w 监视文件变动

server

启动服务器，本地查看

选项 描述
-P 重设端口
-s 只是用静态文件
-l 启动日记记录，使用覆盖记录格式
–drafts 查看草稿

deploy

部署网站到远端服务器

参数 描述
-g 部署之前预生成静态文件

clean

清除缓存文件和已经生成的静态文件。
在某些情况下，尤其是更换主题后，如果发现站点的更换无论如何也不生效，可能需要运行该命令。

list

列出网站资料

调试模式

在终端显示调试信息并记录到debug.log文件中。

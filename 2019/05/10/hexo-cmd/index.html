operation

init

新建一个网站，使用folder指定文件夹，默认为当前文件夹。

new

新建一篇文章，如果没有设置layout则默认使用_config.yml中的default_layout参数。如果标题中使用空格，需要使用引号括起来

draft

It will create a file under the folder source/_drafts.

新建一篇草稿，草稿并不会被更新到服务器端的个人博客中，会在本地文件夹source\_drafts中新建一个md文件

publish

公布草稿为博客

page

为博客添加新的属性（tag,…）

会在相应在source文件夹下生成相应的markdown文件，分别在每个文件头处添加：

添加完属性后需要在文件头出添加相应的标记，或者在布局模板上添加设置，模板的设置如下所示。

generate

生成静态文件

选项 描述
-d 文件生成后部署到网站上
-w 监视文件变动

server

启动服务器，本地查看

选项 描述
-P 重设端口
-s 只是用静态文件
-l 启动日记记录，使用覆盖记录格式
—drafts 查看草稿

deploy

部署网站到远端服务器

参数 描述
-g 部署之前预生成静态文件

clean

清除缓存文件和已经生成的静态文件。
在某些情况下，尤其是更换主题后，如果发现站点的更换无论如何也不生效，可能需要运行该命令。

list

列出网站资料

调试模式

在终端显示调试信息并记录到debug.log文件中。

生成并发布

set skip item for the hexo system

you can find the skip_render on the _config.yml
